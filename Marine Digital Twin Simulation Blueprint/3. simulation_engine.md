## The engine is a loop — and the order of steps matters

The simulation engine is a **time-loop**: every timestep, it runs through five steps in a fixed order, then advances the clock by one timestep (one week, by default), then repeats until the target end date.

The order of the five steps **cannot be changed**. Each step uses outputs from the previous step. If you ran Step 3 before Step 2, you would be calculating biofouling growth using old flow velocity data instead of freshly computed values. The sequence is physically required, not arbitrary.

---

## 5-Step Simulation Cycle

```
STEP 1  Input Update — Environment
        Pull sea temperature, salinity, dissolved O₂ from database.
        Assign fresh values to each zone.
        Inputs: CMEMS satellite data, AIS ship speed
            ↓
STEP 2  CFD Update — Water Flow
        Calculate how fast water moves over each part of the hull.
        Uses: pre-computed flow table indexed by ship speed + zone type
            ↓
STEP 3  Biofouling Growth
        Grow barnacles and biofilm on each zone.
        Factors: temperature, idle time, coating state
        Warm water + harbour = fastest growth; intact coating = 90% suppression
            ↓
STEP 4  Corrosion Update
        Calculate metal loss per zone this timestep.
        Rate uses k value from quantum-precomputed material_table.json
        Coating protection = coating_thickness / max_thickness (0.0 to 1.0)
            ↓
STEP 5  Coating Degradation
        Wear down paint from flow erosion + UV + chemical decay.
        Flip coating_intact flag when threshold crossed.
        ↺ Step 5 output gates Steps 3 and 4 in the NEXT cycle
            ↓
        Compute risk_score, drag, roughness → save snapshot
```

## Step 1 — Input Update (getting fresh environmental data)

Before anything else, the engine needs to know what the ocean conditions are at this particular timestep. This goes first because every subsequent calculation depends on these environmental numbers.

```python
for zone in ship.all_zones:
    zone.local_temperature = env_database.get_temperature(zone.position, current_date)
    zone.local_salinity    = env_database.get_salinity(zone.position, current_date)
    zone.local_oxygen      = env_database.get_oxygen(zone.position, current_date)
# Data source: CMEMS satellite data, preprocessed daily by ETL pipeline
# ship speed also updated here from AIS feed
```

---

## Step 2 — CFD Update (calculating water flow over the hull)

CFD (Computational Fluid Dynamics) calculates how fast water is flowing over each zone. Flow speed directly drives coating erosion in Step 5.

Running a full CFD simulation every timestep would be too slow. Instead, a **pre-computed flow table** is used — computed offline using OpenFOAM, indexed by ship speed and zone type.

```python
# For live simulation: fast table lookup
zone.local_flow_velocity = flow_table[current_ship_speed][zone.zone_type][zone.position]

# For offline deep analysis: actual CFD formula
zone.local_flow_velocity = ship_speed * zone_geometry_factor * turbulence_coefficient

# flow_table pre-computed by OpenFOAM for speeds 0–25 knots, all zone types
```

---

## Step 3 — Biofouling Growth (barnacles and slime)

The growth model is **multiplicative** — start with a base rate, multiply by factors for temperature, harbour idle time, and coating status. Coating factor is the most important: intact antifouling paint suppresses 90% of growth.

```python
# Temperature factor: warmer water = faster growth (Arrhenius relationship)
temp_factor = exp(0.08 * (zone.local_temperature - 15.0))

# Harbour factor: idle ship in port accumulates fouling faster
idle_factor = 1.0 + 0.3 * hours_in_harbour / 24.0

# Coating factor: intact antifouling paint suppresses 90% of fouling
coating_factor = 0.10 if zone.coating_intact else 1.0

# Growth rate and density update
growth_rate           = BASE_FOULING_RATE * temp_factor * idle_factor * coating_factor
zone.barnacle_density += growth_rate * delta_t_days

# Classify fouling stage for the decision engine
if   zone.barnacle_density == 0:  zone.fouling_stage = 'clean'
elif zone.barnacle_density < 10:  zone.fouling_stage = 'slime'
elif zone.barnacle_density < 100: zone.fouling_stage = 'barnacle'
else:                             zone.fouling_stage = 'heavy'
```

---

## Step 4 — Corrosion Update (metal loss)

This step calculates how much metal the zone loses this timestep. The physics is electrochemical corrosion — the reaction of iron with dissolved oxygen in saltwater. The key variable is **k**, the material rate constant, computed by the quantum solver and stored in `material_table.json`.

```python
# Get rate constant k for this alloy, temperature, and salinity
k = material_table[zone.alloy_type][zone.local_temperature][zone.local_salinity]

# Coating protection: 1.0 = fully coated, 0.0 = bare metal
coating_protection = zone.coating_thickness / MAX_COATING_THICKNESS

# Corrosion rate equation (electrochemical model)
zone.corrosion_rate  = k * zone.local_oxygen * salinity_factor * temp_factor \
                         * (1.0 - coating_protection)

# Update cumulative metal loss
zone.corrosion_depth += zone.corrosion_rate * delta_t_years
```

---

## Step 5 — Coating Degradation (paint wears down)

Three mechanisms attack the coating simultaneously every timestep. When `coating_intact` flips to `False`, it changes the rates in Steps 3 and 4 for the **next cycle** — and stays elevated until the coating is repaired. This feedback loop creates the compounding, accelerating degradation seen on real ships that miss maintenance.

```python
# Three coating wear mechanisms
mechanical_loss = EROSION_CONSTANT * zone.local_flow_velocity**2 * delta_t
chemical_loss   = coating_props['chemical_decay_rate'] * delta_t
uv_loss         = coating_props['uv_decay_rate'] * uv_index * delta_t
# uv_loss only applies to zones above the waterline

zone.coating_thickness = max(0.0, zone.coating_thickness
                             - mechanical_loss - chemical_loss - uv_loss)

# Update the gate flag
FAILURE_THRESHOLD   = coating_props['failure_threshold']  # 15 µm default
zone.coating_intact = (zone.coating_thickness > FAILURE_THRESHOLD)

# Compute derived outputs and save snapshot
zone.roughness_index  = zone.corrosion_depth  * roughness_factor \
                      + zone.barnacle_density * barnacle_roughness_factor
zone.drag_coefficient = BASE_DRAG * (1 + zone.barnacle_density * DRAG_PER_BARNACLE)
zone.risk_score       = compute_risk_score(zone)
```

## Layer Dependencies — why you cannot reorder the steps

```
Step 1: Environment  →  provides local_temperature, local_oxygen, local_salinity
            ↓
Step 2: CFD Flow     →  provides local_flow_velocity (uses ship_speed from Step 1)
            ↓
Step 3: Biofouling   →  uses temperature, idle_time, coating_intact (from PREVIOUS cycle)
            ↓
Step 4: Corrosion    →  uses k table, oxygen, salinity, coating_protection
            ↓
Step 5: Coating      →  uses flow_velocity (Step 2), UV data
            ↓
        Updates coating_intact flag
            ↺ FEEDS BACK into Steps 3 and 4 next cycle
            ↓
        Compute risk_score, drag, roughness → save snapshot
```

## Timestep and Time Configuration

| Mode | Timestep | Steps/Year | Best For |
|------|----------|-----------|---------|
| Fast preview | Monthly | 12 | 5–10 year strategic planning |
| Standard (default) | Weekly | 52 | Normal operational forecasting |
| High fidelity | Daily | 365 | Deep investigation of one zone |

Default simulation window: **2 years forward** from current date.
